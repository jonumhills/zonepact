System Design

Two Agents. One Pipeline.

Agent 1 — Pipeline

Crawl → Extract → Write On-Chain

Monitors Wake County's legislative portal for new rezoning filings. Parses each petition — petition number, PIN, zoning change, vote result. Writes every record immutably to ZonePactRegistry on Base Sepolia. Fires an SNS alert to subscribed lenders when a watched parcel is filed.

AWS Bedrock Bedrock Action Groups Amazon SNS Base Sepolia
Agent 2 — Intelligence

Query → Verify → Respond

Accepts natural-language queries: address, PIN, petition number, or zoning type. Looks up the parcel in Supabase (434k records), verifies the petition on-chain, returns cryptographic proof with a BaseScan link. Every verified lookup is saved as a JSON report to Amazon S3.

Agent 1 — Data Ingestion

Pipeline Agent

County zoning records live in legislative portals, PDFs, and OnBase document systems — no API, no standard format. The pipeline agent crawls these sources on a schedule, extracts structured petition data, and writes each record permanently to the ZonePactRegistry smart contract on Base Sepolia. The result: a trustless, immutable on-chain registry that any protocol can query directly.

Crawl → On-Chain flow
1
Scheduled crawl — County legislative portal
Bedrock agent wakes on schedule and issues HTTP requests to Wake County's rezoning petition portal. Detects new filings by comparing petition numbers against the on-chain totalPetitions() count. No official API — agent parses raw HTML and PDF attachments.
2
Extract structured data — Claude action group
Claude reads each petition document and extracts: petition number, parcel PIN, petitioner name, present zoning, proposed zoning, meeting date, vote result, and county. Unstructured legislative text → typed JSON record.
3
Enrich from ArcGIS — parcel geometry + owner
Agent cross-references the PIN against Wake County ArcGIS to pull owner name, site address, land use, assessed value, and polygon geometry. Enriched record is written to Supabase for fast query access by the intelligence agent.
4
Write on-chain — ZonePactRegistry.sol · Base Sepolia
Agent calls registry.addPetition(petitionNumber, pin, petitioner, presentZoning, proposedZoning, status, voteResult, meetingDate, county).
Record is stored immutably on Base Sepolia with a recordedAt UNIX timestamp. Cryptographic proof — tamper-proof, trustless, permanently auditable.
5
Fire SNS alert — per-PIN filter policy
After writing on-chain, agent calls sns.publish() with a MessageAttribute for the PIN. Lenders subscribed via POST /api/watch receive a targeted email alert the same day a petition is filed — not a broadcast, a PIN-matched notification. Aave's liquidation bot subscribes each collateral PIN at loan origination.

Agent 2 — Query & Verify

Intelligence Agent

DeFi lenders query land records in plain English — address, PIN, or petition number. The intelligence agent looks up the parcel in Supabase, verifies the petition on-chain, and returns a structured response with cryptographic proof. Every query costs 0.001 ETH via the x402 protocol, verified on Base Sepolia before the agent runs.

Query flow
1
x402 payment gate — 0.001 ETH · Base Sepolia
Lender sends query to POST /api/chat with X-Payment: {"txHash": "0x..."}. Backend calls eth_getTransactionReceipt + eth_getTransactionByHash: checks recipient address, value ≥ 0.001 ETH, status == 0x1, block age < 5 minutes. Used hashes are tracked to prevent replay. Fails with HTTP 402 if not paid.
2
Claude agent loop — AWS Bedrock · anthropic.claude-opus-4-5
Bedrock agent receives the query and selects the right tool: address or PIN → lookup_parcel, petition number → get_onchain_history, zoning type → filter_by_zoning. Runs a multi-step reasoning loop — up to 12 iterations — until it has enough data to respond.
3
lookup_parcel — Supabase · 434k parcels
lookup_parcel("3420 Quail Hollow Dr", "raleigh_nc")
ILIKE address search against 434k Wake County parcel records. Returns PIN, owner, site address, land use, assessed value, area in acres, and whether the parcel has an active rezoning petition. Polygon geometry returned as GeoJSON — rendered live on Mapbox map.
4
get_onchain_history — ZonePactRegistry · Base Sepolia RPC
registry.getPetition("TCZ-16-23") called directly against the public Base Sepolia RPC. Returns: petitioner, present zoning, proposed zoning, vote result, meeting date, and recordedAt timestamp — cryptographic proof of what was verified and when. BaseScan link included in every response.
5
S3 report upload — Amazon S3 · zonepact-reports/
After every verified lookup, backend uploads a structured JSON report to S3: reports/raleigh_nc/TCZ-16-23-2026-05-07.json.
Contains owner, PIN, zoning change, blockchain proof, BaseScan link. A 7-day presigned download URL is returned in the API response. Centrifuge attaches this URL to their loan NFT metadata.
